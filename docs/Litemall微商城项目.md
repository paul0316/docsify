# Litemall微商城项目













## 电商系统业务架构

分类：

B2B模式：电子商务交易的供需双方都是商家，他们使用了互联网技术或者各种商业平台完成交易。

B2C模式：电子商务交易的供给方是企业，而需求方是个人，比如苏宁易购、唯品会以及小米有品。

C2C模式：个人与个人之间的电子商务，淘宝就是典型的 C2C 的代表，还有ebay、滴滴打车、喜马拉雅和猫眼娱乐等。

O2O模式：将线下的商务机会与互联网结合，让互联网成为线下交易的平台，也就是国内现在经常提起的“新零售”。比如美团外卖、58同城、大润发和美特斯邦威等。



前台

APP/H5/PC

中台

供方（不一定有，比如慕课网）合同、商品、库存、订单、促销（商品，优惠券，订单促销类似拼团）、会员（定制规则）、购物车（数据需要长期保存，MangoDB，商品失效问题，加入购物车的价格和现在的价格）、支付（重要，苹果2016年出现的问题，微信支付的方式）、发票（相较之下容易实现）、价格、店铺（员工的权限）、分享等等

后台

售后（处理退款退单等）、物流、仓储、采购、财务、评价等等
